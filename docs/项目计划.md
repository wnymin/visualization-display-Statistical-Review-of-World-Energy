# 项目计划：世界能源统计可视化网站

## 1. 项目目标
- 构建一个面向公众的能源数据可视化网站，展示《Statistical Review of World Energy》核心指标。
- 基于 Vue 框架完成前端实现，并通过 GitHub Pages 稳定发布。
- 形成可持续更新的数据处理与发布流程，支持后续年度数据迭代。

## 2. 需求分析

### 2.1 业务需求
- 展示全球及主要国家能源相关统计数据。
- 支持按国家、能源类型、年份等维度查看和对比。
- 提供清晰的数据来源和指标解释，避免误读。

### 2.2 功能需求
- 首页：项目介绍、数据来源说明、导航入口。
- 总览页：全球趋势图与关键指标卡片。
- 对比页：多国家或多能源类型对比分析。
- 详情页：单国家或单指标的时间序列分析。
- 筛选与搜索：国家、年份区间、能源类型。

### 2.3 非功能需求
- 性能：首屏加载快，图表交互流畅。
- 可维护性：组件化、可复用、数据逻辑清晰。
- 兼容性：桌面与移动端均可用。
- 可部署性：主分支自动构建并发布到 GitHub Pages。

## 3. 范围与边界
- 已纳入范围：
  - 使用仓库现有数据文件 `EI-Stats-Review-ALL-data.xlsx` 完成展示。
  - 前端展示、交互筛选、部署流程优化。
- 暂不纳入范围：
  - 后端服务与数据库建设。
  - 复杂用户系统（登录、权限、个性化配置）。

## 4. 阶段目标与步骤

## 阶段 A：数据理解与指标设计
- 目标：形成统一指标口径和字段字典。
- 步骤：
  1. 盘点 Excel 中工作表、字段、单位、时间范围。
  2. 定义核心指标清单（如产量、消费量、碳排放等）。
  3. 制定缺失值和异常值处理规则。
  4. 输出《数据字典与口径说明》。
- 交付物：数据字典文档、指标口径说明。

## 阶段 B：信息架构与页面方案
- 目标：确定网站结构、页面层级和交互路径。
- 步骤：
  1. 设计页面地图与导航结构。
  2. 绘制低保真原型，明确各页面展示内容。
  3. 确定图表类型与每个图表的交互方式。
- 交付物：页面结构图、原型稿、图表清单。

## 阶段 C：前端实现与数据接入
- 目标：完成核心页面开发并接入处理后数据。
- 步骤：
  1. 建立页面与组件目录，搭建基础布局。
  2. 实现筛选控件、图表容器与通用交互。
  3. 接入数据文件，完成字段映射与展示。
  4. 增加空状态、错误状态和加载状态。
- 交付物：可运行的核心页面版本。

## 阶段 D：质量优化与验收
- 目标：确保准确性、性能与可用性。
- 步骤：
  1. 对关键图表进行数据抽样核对。
  2. 优化渲染性能和首屏资源加载。
  3. 完成移动端适配与交互细节修正。
  4. 进行构建验证与回归检查。
- 交付物：验收版本与问题清单关闭记录。

## 阶段 E：部署发布与运维
- 目标：形成可持续发布流程。
- 步骤：
  1. 确认 GitHub Actions 构建链路稳定。
  2. 验证 GitHub Pages 路径、静态资源、数据文件访问。
  3. 输出发布说明与更新流程。
  4. 规划后续版本迭代节奏。
- 交付物：线上可访问站点、发布与维护文档。

## 5. 验收标准
- 数据正确性：核心指标抽样核对无明显偏差。
- 功能完整性：总览、对比、详情、筛选均可用。
- 体验可用性：桌面与移动端可读性良好。
- 工程稳定性：本地构建和线上部署持续成功。
- 文档完整性：项目说明、数据口径、部署流程可追溯。

## 6. 风险与应对
- 数据口径不一致：建立字段字典和统一换算规则。
- 图表过多导致性能下降：按需渲染、懒加载、数据裁剪。
- GitHub Pages 部署失败：固定工作流权限并保留手动触发发布。
- 需求变更频繁：采用分阶段交付，优先保证核心路径可用。

## 7. 里程碑建议
- M1：完成数据字典与指标定义。
- M2：完成页面结构与原型确认。
- M3：完成核心页面开发与数据接入。
- M4：完成性能优化与验收。
- M5：完成线上发布与文档沉淀。
